# コトダマ — コンセプト強化レポート

> 並列リサーチエージェントによる UX/競合分析の集約。
> 著者(オーナー)の懸念「自分でも使うか微妙、メモアプリやライブベアーに勝てない」への対応案。

## 1. 市場分析サマリー

| アプリ | コアコンセプト | 良いレビュー | 悪いレビュー |
|---|---|---|---|
| **I am** | 通知でアファメ届ける | 通知の絶妙さ・デザイン | **日本語不自然・誤字・課金圧** |
| **ThinkUp** | 自分の声で録音→再生 | **自声の効果実感・音楽** | 録音面倒・UI複雑・有料化 |
| **Streaks** | 連続日数の美学 | Apple Design賞・シンプル | カスタム不足・有料 |
| **みんチャレ** | 5人匿名チームで強制力 | サボれない圧 | 課金必須・人間関係疲れ |
| **Lifebear** | 手帳+ノート統合 | スタンプ豊富・日記併用 | 重い・Web終了不安 |

## 2. コトダマが取るべき3つのフック (優先順)

### 🥇 第1位: 自分の声で録音 → 朝晩自動再生 (実装中規模)

**根拠**:
- ThinkUp の最大の武器。「自声は他人の声より3倍刺さる」という研究エビデンス (Production Effect の延長)
- 既に Production Effect を中核に据えているコトダマと整合
- AdMob 運用と完全両立、サーバ不要 (録音は端末ローカル)

**実装範囲**:
- AVAudioRecorder で 15-30秒の録音
- 端末ローカル保存 (Documents/recordings/ 配下)
- 通知タップ → アプリ起動 → 自動再生
- 削除/再録音は1タップ
- 録音 UX は Streaks 並みのシンプルさ

**コスト**: 中 (1-2日) — AVFoundation + 既存 NotificationService 拡張

### 🥈 第2位: 連続日数バッジ + 「途切れたら最初から」演出 (実装小規模)

**根拠**:
- Streaks の核心。30-40代男性は「積み上げ可視化」と「損失回避」に強反応 (行動経済学: プロスペクト理論)
- 既に streak フィールドは実装済 → UI 強化のみ

**実装範囲**:
- 連続日数を Hero として目立たせる (現状の小さいバッジから昇格)
- 途切れた時の "もう一度" ダイアログ (責めない / 再開促進)
- 7/14/21/30/100日の節目バッジ獲得アニメ
- ロック画面ウィジェット (Phase 2) で常時可視化

**コスト**: 小 (半日) — UI のみ

### 🥉 第3位: 朝の通勤時刻ピンポイント通知 + ロック画面ウィジェット (実装中規模)

**根拠**:
- I am の強みを純日本語で再現
- 「7:42、◯◯駅入線時刻に自分の言葉」が刺さる (時刻 × 場所のフックは行動経済学的に強い)

**実装範囲**:
- 通知メッセージに自分の登録した言葉を含める (現状はランダム/汎用)
- WidgetKit でロック画面に「今日の言霊」表示
- 簡易: 1日2回固定 → カスタム: 通勤時刻入力 (省略可)

**コスト**: 中 (1日) — WidgetKit Extension

## 3. 捨てるべき / 弱める機能

### ❌ 流すだけタイムライン (Phase 1 で実装済だが)
- メモアプリ/SNSの劣化版になる
- Apple 審査でも UGC で何度もリジェクト
- 個人開発の運用負荷 (24時間以内モデレーション宣言が重い)
- **代替案**: 設定でデフォルトOFF (見せない)、研究エビデンスを「短いファクトカード」として通知に同梱

### ❌ 汎用テンプレ集の拡充
- I am の劣化版になる (「自分で書く」こそコトダマの差別化)
- 現在の6種で十分

### ❌ 多機能化
- メモアプリ・カレンダーアプリと同じ土俵で戦わない
- 「言葉を書いて、声に出して読む。それだけ」という潔さを徹底

## 4. ライブベアー連携の可能性

オーナー自身が Lifebear のカレンダーにアファメ書いている → 「メモ/カレンダーで代替できる」を逆手に取る:

- **iCloud カレンダーへの書き込み** (EventKit): 朝晩の音読時刻を予定として登録
- **ホーム画面の「今日の言葉」が iOS のカレンダー Widget の隣に並ぶ** (WidgetKit + 同寸サイズ)
- ユーザーは Lifebear と並列でコトダマを開く習慣ができる

## 5. オーナーへの問いかけ

> 「自分でも使うか微妙」← この感覚を信頼すべし。

実装前に、以下のどれか **1つだけ** に絞ることを推奨:

A) **「30秒で終わる」徹底**: 通知 → タップ → 自分の声で再生 → 完了。Hero画面は1文のみ。
B) **「積み上げの美しさ」**: 1年カレンダービューで埋まる達成感 (Streaks型)。
C) **「自分の声」アンプリファイア**: ThinkUp 型の中核機能 + 簡易UI。

**個人的おすすめ**: C (自分の声録音) → A (30秒UX) の順。
B は他者との競争性が薄いため Streaks の代替にしかならない。
