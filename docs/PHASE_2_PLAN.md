### 工数

## 🤖 2. サクラbot サーバー版 (Cloud Functions)

> 現セッションでクライアントサイド版は実装済 (`SakuraTemplateProvider.swift`)。
> サーバー版は他デバイスから見える「リアル投稿」として動作させる場合の Phase 2 機能。

### 実装案
- `functions/sakuraSeeder.js` (Firebase Cloud Functions, scheduled)
- 15分毎に実行 (Free tier の月125k呼び出し内)
- 各ロケールの post カウントを取得 → 100件未満 & リアル投稿 5件未満 なら 1-3件追加
- 投稿者UID を `_sakura_001` ~ `_sakura_030` で固定 → ユーザーがブロック可能
- Firestore Security Rules を「`_sakura_*` 投稿は誰でも読み込める」に拡張

### 工数
- 中 (1-2日) — JS 関数 + Firebase deploy + 動作確認
- 運用コスト: $0 (Free tier 内)

## 📅 4. 言葉の期限・曜日・朝夜設定

### ユーザー要望
- 開始日・終了日設定: 4/30まで → 29日まで自動表示、超えたら「含めない」へ移動
- 曜日設定: 月・火...
- 朝のみ・夜のみ
- 含めない理由表示: 「期限切れのため」等

### 行動経済学的根拠
- **WIP制限 (Personal Kanban)**: 同時に追わせるタスクを制限することで完遂率↑
- **コミットメントデバイス (Thaler & Sunstein)**: 期限を区切ることで実行率↑
- **時系列ビュー (Apple Reminders / Things)**: 「今日」「明日」「来週」分類が直感的

### 実装案
- Affirmation モデル拡張: `startDate`, `endDate`, `weekdayMask: Int` (1..127), `slotMask: Int` (morning=1, evening=2)
- HomeView の @Query フィルタ: 現在日時/曜日/スロットでマッチ
- 「含めない」セクションに「期限切れ」「対象外曜日」等の理由バッジ
- 設定画面で「期限切れ自動アーカイブ」のトグル

### 工数
- 中 (1日)

## 📊 8. 別アカウント/別デバイス検証

### ユーザー要望
- ロケーションごとの投稿が他人から見えるか
- Firebase に登録されているか
- 100件超えた時の動作
- ブロックが機能するか

### テスト計画
1. **デバイスA**: 匿名認証 → ja_JP に投稿 → Firestore コンソールで confirm
2. **デバイスB**: 匿名認証 → ja_JP のフィードに A の投稿が出るか
3. **デバイスB**: A の投稿をブロック → 即時非表示確認
4. **デバイスB**: ロケーション切替 → en に切替で別の投稿セット
5. **大量投稿**: 100件投稿 → 古い順に sort されるか、表示が 100件で止まるか
6. **24h経過**: expireAt < now の投稿が表示されないか

### 工数
- 小 (2-3時間) — 実機 2台 or シミュレータ 2台
