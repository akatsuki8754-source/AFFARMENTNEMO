# コトダマ — バグ網羅チェックリスト

> 目的: 「見えているけど機能していない」「タップしても無反応」「ロード中表示が消えない」
> 「タイムアウトで何も起こらない」など、ユーザが**気付きにくい不具合の網羅検出**。
>
> 使い方: 各項目の左側 `[ ]` を `[x]` に変えながら手動 or UITest で確認する。

## 1. オンボーディング (`OnboardingFlowView`)

### スプラッシュ
- [ ] 1秒で自動的にWelcomeへ遷移する (タイマーが動作している)
- [ ] 1秒経っても遷移しない場合に手動進行できる回避策があるか (現状なし → 改善要)
- [ ] アプリアイコン/ロゴが表示されている (`quote.bubble.fill`)
- [ ] ダークモード/ライトモード両方で文字が見える

### Welcome
- [ ] 「ようこそ」タイトルが**中央寄せ**で表示
- [ ] 3つのチェックリストが左寄せで読みやすい
- [ ] 「はじめる」ボタンがタップ可能でキー反応 (グレーアウトしていない)
- [ ] iPhone SE (小画面) で全要素が見える
- [ ] iPad で max-width 制限が効いている
- [ ] 戻るジェスチャーが**無効**になっている (一度進んだら戻れない仕様)

### 最初の言葉
- [ ] テキスト入力欄がフォーカスされる
- [ ] 200文字制限が効いている (201文字目以降が入らない)
- [ ] 文字数カウンタ `0 / 200` が更新される
- [ ] **テンプレ6種が全てタップ可能** (バグ修正済 v4)
  - [ ] 「なりたい自分」: 空入力時 → `私は` 挿入 / 入力ありなら確認ダイアログ
  - [ ] 「叶えたい目標」: 空入力時 → `を達成したい` 挿入
  - [ ] 「大切にしたいこと ⭐」: 空入力時 → `を大切にしたい` 挿入
  - [ ] 「習慣のキッカケ ⭐」: 空入力時 → `もし、したら、する` 挿入
  - [ ] 「自分への励まし」: 空入力時 → `今日も` 挿入
  - [ ] 「ビジネスの計画」: placeholder 空 → 「テンプレートを選択しました」フィードバック
- [ ] テンプレを2回タップで選択解除される
- [ ] テンプレ選択時に挿入フィードバック (緑チェック ✓) が3秒間表示される
- [ ] 同じテンプレを再度タップして解除した時にフィードバックも消える
- [ ] 入力ありの状態で別テンプレタップ → 「置き換え確認」ダイアログ表示
- [ ] 「キャンセル」で何も変わらない / 「置き換える」で挿入される
- [ ] 「次へ」: 空でも進める (`onSubmit` 呼ばれる、空時は保存スキップ)
- [ ] 「スキップ」: 空のまま次へ進む

### 通知時刻設定
- [ ] 朝/夜 のトグルがそれぞれ動作する
- [ ] DatePicker がタップで開き、時刻変更が反映される
- [ ] トグルOFFで DatePicker が disabled になる
- [ ] 「完了」: 通知許可ダイアログ → 拒否してもアプリは動く
- [ ] 通知許可後にローカル通知が登録される (確認は実機)

- [ ] 時間帯で挨拶が変わる (5-11=おはよう / 11-18=こんにちは / 18-5=こんばんは)
- [ ] ストリーク数が正しい (前日読み上げ済 → 連続加算)
- [ ] レベル/XP プログレスバーが計算通り更新

- [ ] 「音読する」が ReadAloudView を fullScreenCover で開く
- [ ] 朝のセット表示が `scheduleMorning && isActive` のみ
- [ ] 各アファメに種別アイコンが付く (🌟/⭐/💪)
- [ ] 0件状態で「言葉を追加してみましょう」CTA が出る
- [ ] 今日のヒントが表示され、`✕` で当日非表示
- [ ] AdMob バナーが画面下に表示 (50pt 高、SDK 未統合時は透明)
- [ ] iPad で max-width 制限が効いている

## 3. アファメ追加/編集 (`AddAffirmationView`)

- [ ] 200文字制限
- [ ] テンプレ選択時の挿入が動作 (オンボーディングと同じロジック)
- [ ] カテゴリ選択ピッカーが動作
- [ ] 朝/夜 トグル
- [ ] 「キャンセル」で破棄、確認ダイアログなし (空入力前提)
- [ ] 「登録する」でホームに戻り、新規アファメが一覧に出現
- [ ] **編集時**: 既存値で初期化されている / 「保存」で更新 / 「削除」で確認ダイアログ
- [ ] 削除後5秒間 SnackBar で Undo 可能

## 4. 音読 (`ReadAloudView`)

- [ ] アファメ文字が大きく中央表示 (28pt)
- [ ] 「声に出して読みましょう」メッセージが見える
- [ ] 「✓ 読みました」ボタンタップで:
  - [ ] パーティクルアニメーション (現在ハート簡易)
  - [ ] readCount +1, lastReadAt 更新
  - [ ] 次のアファメへ自動遷移
  - [ ] 最後なら音読完了画面へ
- [ ] 「🔊 補助読み上げ」: タップで TTS 開始 / 停止 / 連打で重複しない
- [ ] 「⏭ スキップ」: 次のアファメへ (ストリーク加算なし)
- [ ] バツボタン (左上) で一覧に戻る、進行はリセット
- [ ] 全件読み上げ後、ストリーク/XP 加算結果が表示
- [ ] 「ホームへ」で AdMob インタースティシャル判定 → ホームへ

## 5. ライブラリ (`LibraryView`)

- [ ] 各カードに種別アイコン
- [ ] **左スワイプ → 編集** (青ペンアイコン)
- [ ] **右スワイプ → 削除** (赤ゴミ箱)
- [ ] 削除後5秒間 Undo 可能 (戻るボタンで復元)
- [ ] 5秒経過 or 別アクションで物理削除
- [ ] 0件状態で「言葉を追加」CTA
- [ ] 右上 `+` でAddAffirmationView 開く

## 6. 流すだけタイムライン (`TimelineView`)

### EULA ゲート (初回)
- [ ] 「規約を読んで同意する」ボタンが見える
- [ ] お問い合わせリンクが見える
- [ ] EULA モーダル: 最後までスクロールで「同意する」が有効化
- [ ] チェックボックス未チェックで「同意する」disabled
- [ ] 「キャンセル」でゲートに戻る、進めない

### 通常時
- [ ] 投稿一覧が表示 (Firebase 接続時)
- [ ] 各投稿の `⋯` メニューが**タップ可能** (44pt 領域)
  - [ ] 自分の投稿: 「コピー」「削除」が出る
  - [ ] 他人の投稿: 「コピー」「隠す」「報告する」「ブロック」が出る
- [ ] **隠す**: 即座に該当投稿が消える (locallyHidden 反映)
- [ ] **削除**: Firestore で isHidden=true → 一覧から消える
- [ ] **ブロック**: ユーザーUIDの全投稿が消える
- [ ] **報告**: SnackBar で「報告を受け付けました」
- [ ] 上部右の「?」ボタンで ContactView 開く
- [ ] 投稿フローティングボタン (右下): EULA同意済みなら ComposeView
- [ ] Compose:
  - [ ] 100文字制限
  - [ ] URL/メール/電話番号/SNS ID 拒否
  - [ ] NGワード拒否
  - [ ] 5行制限
  - [ ] 「投稿する」disabled if empty
  - [ ] 投稿後ダイアログで成功表示

## 10. 失敗パターン (見えない/反応しない/タイムアウト)

### 「ボタンがあるが見えない」
- [ ] disabled 時の色コントラストが十分 (WCAG AA)
- [ ] AdMob バナー裏に CTA が隠れていない (bottom safe area + 50pt)
- [ ] キーボード表示時に主要ボタンが隠れない (`.scrollDismissesKeyboard(.interactively)`)

### 「タップしても反応しない」
- [ ] テンプレチップの最小タップ領域 44pt (修正済)
- [ ] Menu 内のボタンが破壊的アクションでも反応する
- [ ] Sheet 内の「キャンセル」が常に動く

### 「ロード中で止まる」
- [ ] Firestore 接続失敗時の fallback (オフラインメッセージ)
- [ ] AdMob 広告ロード失敗で UI が止まらない
- [ ] 通知許可ダイアログ拒否後にスタックしない

### 「数秒で消える」
- [ ] スプラッシュ → Welcome 遷移後にスプラッシュが残らない
- [ ] テンプレ挿入フィードバック (3秒) が確実に消える
- [ ] SnackBar Undo (5秒) がカウントダウン後に物理削除

### 「読み込めない」
- [ ] Firebase 未初期化時のクラッシュ防止 (#if canImport)
- [ ] GoogleService-Info.plist 不在時のフォールバック
- [ ] SwiftData マイグレーション失敗時の挙動

## 11. メモリ / パフォーマンス

- [ ] アプリ起動 → ホーム表示まで 2秒以内
- [ ] アファメ100件登録時にスクロールが滑らか
- [ ] 流すだけタイムライン無限スクロールでメモリリークなし
- [ ] バックグラウンド復帰時にビュー再構築されすぎない
